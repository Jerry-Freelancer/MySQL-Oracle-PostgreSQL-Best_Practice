<p align="center">How to modify the MySQL UUID</p>

â€‹        Description of the problem: When the cluster is built, the image of the main service is cloned, so that the service UUIDs of all nodes are consistent. At this time, when adding a node to the cluster, a UUID conflict error will be prompted.



Solution

1. Use the uuid function to generate a new uuid

  ```sql
  mysql> select uuid();
  +--------------------------------------+
  | uuid()                               |
  +--------------------------------------+
  | b33057ff-bec6-11eb-ad94-000c29af6856 |
  +--------------------------------------+
  1 row in set (0.00 sec)
  ```

2. View the configuration file directory

  ```sql
  mysql> show variables like 'datadir';
  +---------------+------------------------+
  | Variable_name | Value                  |
  +---------------+------------------------+
  | datadir       | /mysql/data/3306/data/ |
  +---------------+------------------------+
  1 row in set (0.01 sec)
  ```

3. Edit the configuration file directory

  ```shell
  vi /mysql/data/3306/data/auto.cnf
  
  [auto]
  server-uuid=b33057ff-bec6-11eb-ad94-000c29af6856
  ```

4. Restart the service

  ```shell
  systemctl restart mysqld
  ```
